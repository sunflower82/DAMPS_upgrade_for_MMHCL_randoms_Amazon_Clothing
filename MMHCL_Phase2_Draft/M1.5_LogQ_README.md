## 3. Giao thức sweep M1.5

### 3.1 — Cảnh báo numerical (chi tiết trong docstring Block 2)

Với τ=0,3 và cosine sim ∈ [-1, 1] (model.py:649 `F.normalize`), log_q
ở Amazon Clothing rơi vào khoảng **[-12, -4]**. Subtract trực tiếp rồi
chia cho τ tạo logit hiệu dụng ~40 → `exp(40)` overflow float32.

→ **Phải sweep `logq_scale` trước khi tin spec literal**.

### 3.2 — Sweep matrix (3 seeds × 4 scale × 2 mode = 24 runs ~ 30 GPU-h)

| Cấu hình | `--logq_mode` | `--logq_scale` | `--logq_clip` | Lý do |
|---|---|---|---|---|
| h-laplace-0p05 | laplace | 0,05 | 5,0 | Subtract rất nhẹ |
| h-laplace-0p10 | laplace | 0,10 | 5,0 | Phổ biến trong literature |
| h-laplace-0p30 | laplace | 0,30 | 5,0 | Trung gian |
| h-laplace-1p00 | laplace | 1,00 | 5,0 | Spec rev53 literal |
| h-sqrt-0p10 | sqrt | 0,10 | 5,0 | Less-aggressive DGRec |
| h-sqrt-1p00 | sqrt | 1,00 | 5,0 | Spec DGRec literal |

Mỗi cấu hình chạy seed ∈ {42, 43, 44}. Báo cáo mean ± std cho R@20 và
NDCG@20. β giữ cố định = 1,0 trong sweep này (β là chiều thứ cấp, sweep
sau khi đã chốt scale).

### 3.3 — Tiêu chí pass M1.5

Lấy `--logq_scale` thắng cuộc trên R@20 trung bình của 3 seed. Chạy thêm
5 seed (45–49) ở chính cấu hình đó để đủ 5 seed như rev45. Pass nếu:

1. **R@20** trung bình ≥ **0,0925** (tức ≥ +1,5% so với rev45 0,0909).
2. Không seed nào < **0,0890** (rollback gate, rev53 §6 line 204).
3. NDCG@20 trung bình **không giảm** so với rev45 0,0416.
4. Welch's t-test 2-tailed cho R@20: p < 0,05.

Nếu (1) hoặc (2) fail → **rollback**: tắt `--enable_logq`, đóng nhánh
LogQ, ghi nhận negative result rồi chuyển sang M1 (SimGCL).

## 4. Bẫy đã biết

### 4.1 — log_q bị quên không gọi setter
Buffer log_q được khởi tạo zeros. Nếu `enable_logq=True` nhưng quên gọi
`set_log_q`, code sẽ silently chạy như baseline. Block (4) ngăn việc này
bằng `RuntimeError` khi `float(self.log_q.abs().sum()) == 0.0`.

### 4.2 — Áp LogQ lên user branch
LogQ là **item popularity prior** — bản chất là sample correction cho
phân phối item. Áp lên `bcl_user` (user branch) sẽ double-count user
activity bias. Block (5) cứng `apply_logq=False` cho `bcl_user`.

### 4.3 — Cache stale khi đổi split
Cache file được key bằng SHA1 của (mode, β, n_items, Σcounts). Nếu bạn
chuyển dataset (clothing → sports) mà quên xoá cache, signature sẽ
mismatch và cache tự rebuild. Đã có unit test T9 cover trường hợp này.

### 4.4 — Multi-seed parallel write race
Khi 3 seed cùng chạy trên cùng dataset directory với cache lần đầu
chưa tồn tại, cả 3 process có thể đồng thời gọi `torch.save` → race.
**Trước khi bắt đầu sweep**, chạy 1 lần `--enable_logq 1 --epoch 0`
trên seed 42 để materialise cache; các seed sau sẽ load thay vì rebuild.

### 4.5 — log_q trên multi-GPU
Buffer được register qua `register_buffer`, nên `.to(device)` và DDP
broadcast sẽ tự xử lý. Không cần code thêm cho phase multi-GPU sau này.
